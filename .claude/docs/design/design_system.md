# Design system

> Projet: LeaseAI mobile app MVP complet  
> Positionnement: AI-native IT leasing operator  
> Usage: document source-of-truth pour product, design, mobile, backend, risk, ops et demo.

## Direction artistique

LeaseAI combine trois idées visuelles:

- **Capital-first**: sérieux financier, structure, contrôle.
- **AI-native**: intelligence, flux, automatisation.
- **IT lifecycle**: actifs, contrats, renouvellement, circulation.

La DA doit être claire, premium, blanche, bleue, respirante. Les écrans doivent inspirer confiance et vitesse, jamais gadget.

## Typographie

### Primary

Satoshi.

- H1: 32-40 mobile selon écran.
- H2: 24-28.
- Section: 18-20.
- Body: 15-16.
- Small: 12-13.

### Data

IBM Plex Mono.

Usage:

- montants;
- IDs;
- scores;
- dates importantes;
- pourcentages.

## Formes

- Radius cartes: 16-20.
- Radius boutons: 12-14.
- Radius inputs: 12.
- Radius badges: 999.

## Iconographie

Style line icon, simple, précis.

Icônes clés:

- home;
- deal/file;
- document;
- chart;
- payment/card;
- asset/device;
- user;
- settings;
- bell;
- AI/spark;
- shield/check;
- warning.

## Visual elements

- Orbites et flux pour lifecycle.
- Nœuds et connexions pour intelligence.
- Data overlays pour finance.
- Charts simples pour portefeuille.
- Cartes clean pour informations critiques.

## Règle de composition

Une seule hiérarchie claire par écran. Pas de dashboards trop denses sur mobile. Les KPI doivent être grands, lisibles et actionnables.
