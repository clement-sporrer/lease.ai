# Refinancer package

> Projet: LeaseAI mobile app MVP complet  
> Positionnement: AI-native IT leasing operator  
> Usage: document source-of-truth pour product, design, mobile, backend, risk, ops et demo.

## Objectif

Le package financeur doit permettre une décision rapide, standardisée et traçable.

## Contenu du package

### 1. Cover summary

- Deal ID.
- Date.
- Client.
- SIREN.
- Revendeur.
- Montant financé.
- Durée.
- Mensualité proposée.
- Score band.
- Décision demandée.

### 2. Company profile

- Raison sociale.
- Adresse.
- Activité.
- Ancienneté.
- Statut actif.
- Dirigeant/signataire.
- Données enrichies.

### 3. Asset summary

- Catégories matériel.
- Fournisseur.
- Montant.
- Quantités.
- Usage attendu.
- Valeur résiduelle indicative.

### 4. Pricing summary

- Montant financé.
- Durée.
- Mensualité.
- Hypothèses.
- Marge.
- Frais.

### 5. Risk summary

- Score.
- Band.
- Flags.
- Règles appliquées.
- Recommandation.

### 6. Documents

- Devis.
- Extraits entreprise.
- Pièces signataire si disponibles.
- Autres pièces.

### 7. Decision box

- Accord.
- Refus.
- Clarification.
- Commentaire.

## Formats

- PDF synthétique.
- ZIP documents.
- Vue mobile financeur.

## Actions financeur

- `Approve`.
- `Reject`.
- `Request clarification`.

## Audit

À tracer:

- package generated;
- package assigned;
- package viewed;
- document downloaded;
- decision submitted;
- comment added.
