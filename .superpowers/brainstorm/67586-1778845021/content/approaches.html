3 approches pour la roadmap demo

Objectif : "C'est complet — tout le cycle en un seul outil" — 2–4 semaines — démo guidée multi-profil

A

Le récit linéaire

Un seul dossier, pré-seedé, qui traverse toutes les étapes dans l'ordre. Tu changes de rôle au fil de la démo : partner → admin → financier → ops → CFO. Chaque expert voit son moment.

Pour

  • Raconte naturellement "tout le cycle"
  • Facile à suivre pour un public non-technique
  • Chaque profil a un moment fort

Contre

  • Nécessite des écrans à tous les stades
  • Ambitieux en 2–4 semaines
  • Backend à construire pour refi → CFO

Focus : ~7 écrans clés, tous en lecture sur données seedées

B

3 rôles en profondeur

Pick 3 rôles ultra-décisionnels (Admin/Risk + Financier + CFO), construis chacun en profondeur avec des données riches. Les autres rôles sont mentionnés mais pas montrés.

Pour

  • Moins d'écrans = plus de polish
  • Faisable confortablement en 2–4 semaines
  • Chaque écran est impressionnant

Contre

  • Moins "complet" — contredit l'objectif B
  • Les experts ops/revendeurs ne se voient pas dans l'outil

Focus : 3 rôles × 3–4 écrans chacun, très polished

C

Le récit linéaire avec raccourcis stratégiques ★ Recommandé

Variante de A : données pré-seedées à chaque stade, écrans en lecture seule (pas de formulaires pendant la démo). On construit d'abord les vues les plus impressionnantes, on "saute" les étapes à faible valeur visuelle.

Pour

  • Raconte "tout le cycle" — objectif B atteint
  • Faisable en 2–4 semaines (pas de forms complexes)
  • Aucun risque pendant la démo — données fixes
  • Backend simplifié : endpoints GET suffisent

Contre

  • Moins interactif (mais la démo est guidée de toute façon)
  • Il faut choisir les raccourcis avec soin

Focus : 5 rôles × 1–2 écrans clés chacun, données seedées réalistes, navigation fluide
